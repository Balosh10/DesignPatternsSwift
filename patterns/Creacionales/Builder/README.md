## Teoría

El patrón Builder separa la construcción de un objeto complejo de su representación, de manera que el mismo proceso de construcción puede crear diferentes representaciones. Es útil cuando un objeto requiere múltiples pasos de configuración o cuando quieres una API fluida (fluent API) para crear instancias.

**Cuándo usarlo:** cuando la creación de un objeto es compleja o tiene muchas opciones, o cuando quieres evitar constructores con muchos parámetros.
