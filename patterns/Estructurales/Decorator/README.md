# Decorator

Patrón Estructural — Decorator

Descripción

Ejemplo didáctico del patrón Decorator con un playground demostrativo.

---

## Teoría

Decorator permite añadir responsabilidades a un objeto de forma dinámica sin usar subclases. Es útil para componer comportamientos adicionales en tiempo de ejecución y mantener clases pequeñas y enfocadas.

**Cuándo usarlo:** cuando quieras extender comportamiento sin proliferar subclases ni modificar código existente.
